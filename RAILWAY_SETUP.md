# Настройка Railway для Discord бота

## Шаг 1: Создание проекта на Railway

1. Зайдите на [railway.app](https://railway.app)
2. Нажмите "New Project"
3. Выберите "Deploy from GitHub repo"
4. Выберите ваш репозиторий

## Шаг 2: Настройка переменных окружения

В разделе "Variables" добавьте:

```
DISCORD_TOKEN=ваш_токен_бота
PORT=5000
```

## Шаг 3: ВАЖНО! Настройка Volume для сохранения базы данных

⚠️ **БЕЗ ЭТОГО ШАГА ВСЕ ДАННЫЕ БУДУТ ТЕРЯТЬСЯ ПРИ КАЖДОМ ДЕПЛОЕ!**

1. Откройте ваш сервис на Railway
2. Перейдите в раздел "Settings"
3. Найдите секцию "Volumes"
4. Нажмите "Add Volume"
5. Настройте Volume:
   - **Mount Path**: `/app/data`
   - **Name**: `vanguard-data` (или любое другое имя)
6. Нажмите "Add"

## Шаг 4: Обновление кода для использования Volume

Измените в `database.py` путь к базе данных:

```python
# Вместо:
DB_FILE = "vanguard.db"

# Используйте:
import os
DB_FILE = os.getenv("DB_PATH", "vanguard.db")
```

И добавьте в переменные окружения Railway:
```
DB_PATH=/app/data/vanguard.db
```

## Шаг 5: Настройка домена

1. В разделе "Settings" найдите "Networking"
2. Нажмите "Generate Domain"
3. Скопируйте сгенерированный домен (например, `your-app.up.railway.app`)
4. Используйте этот домен для доступа к веб-панели

## Шаг 6: Проверка работы

1. Откройте логи деплоя
2. Убедитесь, что видите сообщения:
   ```
   ✅ База данных инициализирована
   🌐 Веб-сервер запущен на http://0.0.0.0:5000
   Бот запущен как YourBot#1234
   ```

## Миграция данных из JSON в SQLite

Если у вас уже есть данные в JSON файлах:

1. Запустите локально: `python migrate_to_db.py`
2. Загрузите `vanguard.db` в репозиторий
3. Задеплойте на Railway
4. Данные будут сохранены в Volume

## Проверка сохранения данных

После настройки Volume:

1. Добавьте кого-то в вайтлист через панель
2. Сделайте редеплой (Settings → Redeploy)
3. Проверьте, что данные остались

Если данные пропали - проверьте настройку Volume!

## Важные замечания

- ⚠️ Volume обязателен для сохранения данных между деплоями
- 📁 База данных `vanguard.db` должна быть в репозитории (не в `.gitignore`)
- 🔄 При первом деплое база создастся автоматически
- 💾 Все изменения сохраняются в реальном времени в SQLite
