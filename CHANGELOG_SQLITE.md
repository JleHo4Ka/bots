## Как это работает

### Раньше (JSON):
1. Данные загружались из JSON при старте
2. Хранились в памяти
3. Сохранялись в JSON при каждом изменении через `save_data()`
4. ❌ При редеплое на Railway JSON файлы терялись (были в .gitignore)

### Теперь (SQLite):
1. Данные загружаются из БД при каждом обращении
2. Сохраняются в БД автоматически при изменении
3. ✅ База данных в репозитории + Railway Volume = данные сохраняются
4. ✅ Быстрее и надёжнее

## Тестирование

Проверьте:
1. Запуск бота без ошибок
2. JSON файлы НЕ создаются
3. Добавление/удаление из вайтлиста работает
4. Выдача/снятие доступов работает
5. Баны работают
6. Веб-панель работает
7. После перезапуска бота данные сохраняются

## Для Railway

Обязательно настройте Volume:
- Mount Path: `/app/data`
- Переменная окружения: `DB_PATH=/app/data/vanguard.db`

Без Volume данные будут теряться при каждом деплое!
